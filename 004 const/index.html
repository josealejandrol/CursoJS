const

Nos permite crear variables que no vayan a cambiar su valor durante el flujo de nuestra aplicación.

No se puede declarar constantes vacías o sín valor

Una variable const no se puede reasignar su valor.


Una variable const no se puede cambiar su valor cuando es un tipo de dato primitivo, pero cuando es un tipo de dato compuesto esto cambia.


Un objeto es una colección de atributo/clave y valor.


Un array/arreglo es una variable que almacena varias variables


Una constante no funciona de la misma manera para valores primivos que para valores compuestos.

Si a lo largo del flujo de tu aplicación o programa la variable va a cambiar, es decir, puede que empezo con una cadena de texto, number u otro tipo usa let.

Pero si va a ser una variable que en el flujo no va a cambiar entonces utiliza una constante.

Conclusión, olvidate de var en vez de var usa let o const.
